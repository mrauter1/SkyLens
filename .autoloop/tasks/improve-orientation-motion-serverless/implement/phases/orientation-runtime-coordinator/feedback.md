- IMP-001 `blocking` — `SkyLensServerless/lib/sensors/orientation.ts`, `chooseProviderCandidate()`, `selectionCandidates`, `PROVIDER_PRIORITY`: the coordinator collapses multiple distinct event paths into the same public `deviceorientation-absolute` / `deviceorientation-relative` keys and priorities. That means the required ladder order between `deviceorientationabsolute`, `deviceorientation` with `absolute === true`, and the lower-ranked Safari validated compass path cannot be represented; whichever sample writes the shared map entry last wins. Concrete failure: if a real `deviceorientationabsolute` stream and a validated Safari compass-backed stream are both present, the lower-ranked validated path can overwrite the stronger provider and be selected nondeterministically, which violates the spec’s exact provider order. Minimal fix: keep a separate internal provider identity / priority for each ladder rung and arbitrate on that internal key while still emitting the coarse public source labels.
- IMP-002 `blocking` — `SkyLensServerless/lib/sensors/orientation.ts`, `startOrientationSensorProvider()`: sensor startup now awaits three Permissions API queries before constructing the sensor, and the arbitration window still expires after 500 ms. On browsers where `permissions.query()` is slow or stalls, the high-priority absolute sensor can miss the startup window and the coordinator will incorrectly select a lower-priority event/relative provider or fail startup altogether. That is a compatibility regression against the source requirement that permission hints only suppress startup on explicit `denied` results and otherwise must not block working providers. Minimal fix: do not put the awaited permission-hint path on the critical startup path; either race/time-box the queries and treat slow/unsupported results as non-blocking, or start the sensor immediately unless a prompt hint has already come back as explicitly denied.
- IMP-001 follow-up `non-blocking` — Re-reviewed in cycle 2. The coordinator now keeps separate internal event-provider identities/priorities (`deviceorientationabsolute`, plain `deviceorientation` absolute, Safari validated compass absolute, and relative events) while preserving the coarse public `OrientationSource` labels, so the prior ladder-collision issue is resolved.
- IMP-002 follow-up `non-blocking` — Re-reviewed in cycle 2. Sensor permission hints are now time-boxed off the critical startup path, and the mocked test coverage adds the slow-`permissions.query()` case so a delayed Permissions API response no longer costs the absolute sensor the 500 ms arbitration window.
