- IMP-001 `blocking` [SkyLensServerless/components/viewer/viewer-shell.tsx:715]: `getAlignmentFallbackFocusTarget()` is not surface-aware. On mobile it prefers `desktop-align-action` and then the first `settings-sheet` node in the document before the mobile viewer reopen controls. In the existing mobile flow where alignment is opened from the mobile settings path and the user reopens the viewer while alignment remains open, dismissing the alignment sheet can restore focus to a hidden desktop button instead of a visible mobile control. That violates AC-4 because focus does not reliably land on a surviving control that reopens the same surface. Minimal fix: split mobile and desktop fallback resolution, or otherwise filter by the current surface/visibility so the mobile path prefers visible mobile controls (`mobile-align-action`, the mobile settings trigger inside the active overlay, then `mobile-viewer-overlay-trigger`) and excludes hidden desktop actions.
